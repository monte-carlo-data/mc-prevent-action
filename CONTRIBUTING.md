# Contributing to MC Prevent — GitHub Action

Welcome! We appreciate contributions from both Monte Carlo engineers and the community.

## Getting started

1. Fork and clone the repository.
2. Review `action.yml` to understand the action's inputs, outputs, and entry point.
3. Make your changes on a feature branch.

## Pull request guidelines

- One logical change per PR.
- Include a clear description of what changed and why.
- For bug fixes: describe the incorrect behavior and how to reproduce.
- Test your changes by running the action in a workflow before submitting.

## Reporting issues

Please open a GitHub Issue for bugs or feature requests. For security vulnerabilities, see [SECURITY.md](SECURITY.md).
